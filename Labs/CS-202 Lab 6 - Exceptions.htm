Part I - The Exception Classes

Create the following custom exception classes. Review the section in the book on how to create custom exceptions.

  • RadiusCannotBeNegativeException
  • InputNotANumberException
  • InvalidClassStatusException
  • InvalidGpaRangeException

Part II The Circle Class

This should be easy to give you practice using a simple custom exception. Design a class to represent a circle. The constructor should take a radius. The only instance variable should be the radius. Add a getter and setter for the radius. Also add a method which calculates the area of the circle (PI*radius*radius). The setter for radius should throw a RadiusCannotBeNegativeException if the radius that the user is trying to set is less than 0. Be sure to use this method in your Constructor when you are setting the value of the instance variable.

Demonstrate how your circle class works asking the user for input for the radius. If I type in a negative radius I should see the exception, otherwise you should demonstrate the rest of the functionality of your class.

Also, if the user types in an input which is not a number then you should throw an InputNotANumberException

Part III The Student Class

Reuse your Student class from Lab 4 figure out how to enhance it with Exceptions. You should add an instance variable for the student's name and remove anything linking the Student class to the Person class. We will not be using the Person->Student parent/child relationship. Also you may take out the list of Courses. Update your student class so that the CIN is stored as a long type, not a String or int.

The CIN for a Student can trigger an InputNotANumberException.if the value entered for the CIN is not a number.

The Class Status should be "freshman", "sophomore", "junior", "senior", or "graduate" and can trigger an InvalidClassStatusException if at any point the status is changed to something other than the above.

The GPA can trigger an InputNotANumberException and if the value of the GPA is not between the range of 0.0 and 4.0 then it can trigger an InvalidGpaRangeException.

You should be sure to throw the specifc exceptions at any point in your program where it would be possible to trigger that exception, i.e. when an object is created, when the value of the object changes, if the result of a math calculation could trigger an exception, etc.

The Driver

Include a Driver class and thoroughly demonstrate how your classes work with the exceptions. Create a menu to allow the user to create and work with the Circle and Person objects. Your program should loop until the user chooses to exit the program. Also, even if an exception is thrown, you should figure out how to keep your program running without crashing due to the runtime error caused by the exception.
